# Travailler avec la mémoire

Ce tutoriel vous montre comment activer et utiliser le système de mémoire de DeerFlow pour que l'agent se souvienne des informations importantes vous concernant entre plusieurs sessions.

## Activer la mémoire

Dans `config.yaml` :

```yaml
memory:
  enabled: true
  injection_enabled: true
  max_injection_tokens: 2000
  debounce_seconds: 30
```

## Comment fonctionne la mémoire

La mémoire fonctionne automatiquement via `MemoryMiddleware` :

1. **Première conversation** : dites à l'agent vos préférences, votre projet ou votre contexte.
2. **Apprentissage automatique** : l'agent extrait et sauvegarde les faits importants en arrière-plan.
3. **Conversations futures** : les faits de mémoire sont automatiquement injectés dans le prompt système — vous n'avez pas besoin de répéter le contexte.

## Exemple

**Première conversation :**

```
I am a Python backend developer primarily using FastAPI and PostgreSQL.
My team follows PEP 8 and prefers type annotations everywhere.
Please remember this for future code suggestions.
```

**Conversation suivante** (pas besoin de répéter le contexte) :

```
Help me write a user authentication module
```

L'agent produira automatiquement du code de style FastAPI avec des annotations de type.

## Inspecter la mémoire

La mémoire est stockée dans `backend/.deer-flow/memory.json` :

```bash
cat backend/.deer-flow/memory.json
```

## Mémoire par agent

Quand un agent personnalisé est actif, il maintient son propre fichier de mémoire à l'emplacement :

```
backend/.deer-flow/agents/{agent_name}/memory.json
```

Cela permet à chaque agent de conserver ses connaissances apprises de manière séparée.
